Включать/Выключать белый список можно на время и с указанием причины, сервер автоматически включит/выключит его после завершения времени. Игроков также можно добавлять на время.

Команду для добавления/удаления можно использовать как с именем `/whitelistplayer add TheFaser`, так и с uuid `/whitelistplayer add cdc5b40e-103a-403b-9f4e-4b58cb02b9f2`.

При добавлении игрока, который ещё не играл на сервере, в базе данных будет создана запись с указанным `name` и `uuid`. Если сервер официальный, то `name` и `uuid` автоматически создадутся верными. Если сервер неофициальный, то будет использовано введёное значение и пустое, для того чтобы обновиться при первом входе игрока. Вы можете получить предупреждение, что `uuid` или `name` изменились при входе, это нужно проигнорировать

### `empty`

Сообщение, если в белом списке никого нет

### `null_page`

Сообщение, если введённая страница не существует

### `null_player`

Сообщение, если введённый игрок не найден

### `null_type`

Сообщение, если введённый тип неправильный

### `reasons`

Список с ключами и значениями, где ключом является слово, а значением конкретная причина

<Callout type="info" title="Информация">
    Можно вписывать свои причины, например

    ```yaml
    random_kek: "Random reason"
    ```
    Тогда если я напишу `/whitelistplayer add Notch 1d random_kek`, то причиной будет `Random reason`

    Если причина не указана, будет использоваться `default`
</Callout>

### `already_add`

Сообщение, если введённый игрок уже есть в белом списке

### `already_off`

Сообщение, если белый список уже выключен

### `already_on`

Сообщение, если белый список уже включен

### `already_remove`

Сообщение, если введённого игрока нет в белом списке

### `format_add`

Сообщение при добавлении игрока в белый список

### `format_off`

Сообщение при выключении белого списка

### `format_on`

Сообщение при включении белого списка

### `format_remove`

Сообщение при удалении игрока из белого списка

### `person`

Сообщение для игрока, когда он пытается зайти на сервер

### `connection_attempt`

Сообщение для администраторов, когда игрок пытается зайти на сервер и его нет в белом списке

### `auto_add`

Добавлять ли игроков, которые сейчас на сервере, автоматически в белый список

### `auto_add_duration`

Сколько должен наиграть игрок, чтобы его автоматически добавило в белый список. Будет работать только если `auto_add: true`

### `check_duplicate`

Будет ли проверяться, что игрок уже есть в белом списке при добавлении новой записи. Если выключено, то можно будет несколько раз добавлять одного и того же игрока в белый список
