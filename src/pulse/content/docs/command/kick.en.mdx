### `null_player`

Message if the entered player is not found

### `reasons`

A list with keys and values, where the key is a word and the value is a specific reason

<Callout type="info" title="You can add your own reasons, for example">
    ```yaml
    random_kek: "Random reason"
    ```
    Then if I type `/kick player random_kek`, the reason will be `Random reason`

    If no reason is specified, `default` will be used

</Callout>

### `server`

Message for everyone

### `person`

Message for the player
